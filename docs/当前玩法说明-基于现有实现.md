# 当前玩法说明（基于现有实现）

**用途：** 按现有代码与流程，描述用户从打开 App 到在 Room 里与 Mobi 互动的**完整可玩内容**。

**维护规则：** 凡**玩法、流程、可玩功能**有增删改（包括但不限于：新交互、新入口、新阶段、新 Debug、设置/登录变更），**必须同步更新本文档**。工作流见 `.cursor/rules/mvp-plan-workflow.mdc`。

**维护：** 与 [MVP-Phase-Plan](MVP-Phase-Plan.md)、[Mobi交互行为完整设计](Mobi交互行为完整设计.md) 同步。

---

## 一、整体流程

```
每次启动 → AuthView（选择/注册 ID，副标题「每次启动请选择或输入要使用的 ID」）
    ↓
选择或注册账户后 → 按该用户是否已完成 Anima 分支：
    ├─ 未完成 Anima → GenesisCoordinatorView（15 轮 Anima + 转场）
    └─ 已完成 Anima → RoomContainerView（First Room，与 Mobi 相伴）
```

- **每次启动选择 ID**：冷启动时**始终**先显示 AuthView，不沿用上次用户；门控出现时清空当前持久化 userId，用户必须选择已有账户或注册新账户后才能进入，避免搞混不同 Mobi。
- **多账号**：可注册多个账户（用户名 + 自动生成 ID），登录时选择账户；每个账户的 Anima 完成状态、进化、记忆、画像按账号隔离。
- **入口**：`showAuthGate = true`（每次启动）；`showRoom` = 当前用户已完成 Anima，否则走 Genesis。
- **首次进 Room**：若未签署 guardian，先弹 **Kuro 开场介绍（welcomeIntro）** 三屏，内嵌麦克风权限；替代原 guardian 全屏弹窗。

## 二、登录 / 注册（AuthView）

| 玩法 | 说明 |
|------|------|
| **触发时机** | **每次 App 冷启动**都会先显示本页；不记住上次用户，需本页选择或注册后再进入。 |
| **注册** | 输入用户名 → 自动生成 `user_xxxxxxxx` → 加入已注册列表 → 进入游戏（视为当前用户） |
| **登录** | 从已注册列表选一个账户 → 设为当前用户 → 进入游戏 |
| **效果** | 进入后若该用户未完成 Anima 则进入 Genesis；已完成则直接进 Room。选择/切换账号时会清空引擎内用户专属状态（Mobi 外形/人设），进化与记忆按该 ID 加载，对应进入该 ID 的 Mobi。 |

## 三、Anima 阶段（GenesisCoordinatorView）

**形态：** 以太流体 Orb（AminaFluidView），15 轮「创造仪式」对话，全程语音与 Orb 视觉反馈。

### 3.1 对话与流程

| 玩法 | 说明 |
|------|------|
| **语音对话** | 用户说话 → Orb 进入 listening（略放大、视觉反馈）；AI 回复 → TTS 播放，Orb 随 TTS 脉动（visualScale、Core Flutter） |
| **15 轮弧** | Gathering 1–5 / Building 6–12 / Farewell 13–15；每轮有 Soul Hook 引导（Yes-And、承接再抛出）；第 15 轮告别后触发转场 |
| **Magic Trick 协议** | 神秘、反应式语气；噪声/攻击/任务类输入被转为感官 Yes-And |
| **强模型侧写** | 第 13 轮前后可提前 commit：transcript → StrongModelSoulService → visual_dna + persona + memories；无 transcript 时 SoulProfile → GeminiVisualDNAService → DNA；失败则本地 PersonalityToDNAMapper 推导 |

### 3.2 视觉与触觉

| 玩法 | 说明 |
|------|------|
| **Orb 随输入变化** | 用户每轮输入驱动 fluidColor / emotionColors（fromWarmthAndTurn 等），形态与颜色随对话变化 |
| **膝跳反射** | 关键词（冷/火/沉等）→ fluidTurbulence、fluidColor、reflexOffsetY 等瞬时视觉 |
| **触摸** | 按下 Orb 区域 → touchScale 0.9 |
| **Lure（长时间无互动）** | idle > 6s 脉动环；> 15s 每 2s 轻 haptic + HumSound；> 30s 发隐藏指令 + Mystery Purple，引导用户开口 |
| **Entropy** | 对话多样性经接口AI DeepSeek R1 算 entropy，参与 MobiSeed 的 ΔSurprise |
| **背景乐** | GenesisAmbient 循环；Mood DSP（neutral/happy/sad 等）可调 |

### 3.3 转场与诞生

| 玩法 | 说明 |
|------|------|
| **Incarnation 转场** | 第 15 轮结束后出现 IncarnationTransitionView：约 0–10s 耀斑/消散/黑屏，10s stopAmbient + 0.5s 后 Boom，10–23s 播放 genesis_transition.mp4（缺失时 GenesisVideoFallbackView 径向光晕） |
| **Cosmic Sneeze** | 23–28.5s：Void → Spark → Materialization → Sneeze → Connection；各阶段有 sfx + 触觉反馈；onComplete 后进入 Room |
| **诞生结果** | 写入 engine：resolvedMobiConfig（颜色）、resolvedVisualDNA（眼型/耳型/身型等）、roomPersonaPrompt（SoulProfile 人设）；供 Room 阶段使用 |

### 3.4 调试（Debug 浮层）

| 玩法 | 说明 |
|------|------|
| **▲ Debug** | 展开后可见 State / Stage / Doubao 连接 / Soul turn·stage·trait·color·erosion |
| **T/E/S 滑条** | 实时调 Orb 的 Warmth / Energy / Chaos（仅 Anima 有 psycheModel 时） |
| **跳到最后两轮** | interactionCount = 13，再聊两轮即告别 |
| **跳 Mobi newborn/child/adult** | 直接进 Room 并设对应进化阶段（fallback config + persona），跳过 Anima |
| **设置** | 打开 SettingsView（见下） |

## 四、Room 阶段（RoomContainerView）

**形态：** First Room；背景为视差（程序化或 HomeBackground 图）；Mobi 为程序化外形（16 眼型/耳型/身型、人格槽、Soul Vessel）或 CartoonMobi 图；昼夜叠加层（6–18 点为昼，22–6 点房间更暗）。**晚安流程**：22 点后进入 Room 时，约 4s 后启动晚安 flow：Mobi 说晚安 → 钻被（缩小下移）→ 屏幕渐暗。

### 4.0 Kuro 规则守护者（入口与流程）

| 玩法 | 说明 |
|------|------|
| **巢穴** | Room 左下角不显眼的深色区域，**长按约 0.9s** 召唤 Kuro → 行政 overlay：「更改档案」打开设置、「销毁样本」二次确认后重置至创世并重连 Doubao |
| **布置** | 右上角「布置」按钮 → RoomDecorView：5 件家具（水杯/抱枕/画框/地毯/灯）选择后拖拽到房间内放置；放置后 Mobi 看向新物体（lookAtFurnitureOverride）约 3 秒，可自然触发「教这个词」；位置持久化到 UserDefaults |
| **Kuro 开场介绍（Day 1）** | 首次进 Room 未签署协议时，Kuro **welcomeIntro** 三屏：①「这是 Mobi，刚从以太里来，还不会说话」②「教它说话。布置它的房间。它是你的了」③「需要麦克风权限才能教它」[同意][稍后]；同意后请求麦克风权限并关闭；稍后则关闭但不开麦（仍可触摸 Mobi，无语音） |
| **进化考核** | 人格槽（灵器）满 100% 且未满溢过时，先弹 Kuro 考核 overlay（认知/情感模块文案），用户点击「进化许可」后进入裂纹→炸裂→融入序列 |
| **精力与能量账单** | Room 内精力随时间与每轮对话扣减；归零时 Mobi 瘫软（缩小、下沉），并自动弹 Kuro 能量账单 overlay；点击「注入能量」调起 StoreKit 2 购买（未配置时 stub 加满）；购买成功后展示「交易愉快」并关闭，Mobi 恢复 |
| **机制介绍（轻量提示）** | Kuro 在**情境触发**时以轻量浮层（小 Kuro + 单句气泡）宣读一句规则感台词，约 4s 自动消失或轻触消失，每类仅展示一次：**灵器** 首次长按灵器并关闭 Soul Sync 后；**Seeking** 首次沉默约 12s 后 Mobi 主动说话后；**日记** 首次打开并关闭记忆日记后；**精力预警** 精力首次低于约 30% 时；**晚安** 首次 22 点后进房且晚安流程进入 drawing 时。**巢穴入口** 在监护人协议或进化考核结尾顺带一句「若有行政需求，在房间角落长按即可召唤我」。 |
| **库洛语与语音** | Kuro 使用「库洛语」gibberish（音节表按 key 确定性生成）+ 代码合成程序化语音播放（无 TTS、无 WAV 资源）；全屏 overlay 与轻量浮层出现时自动播放，关闭时停播；气泡主句为库洛语、副句为中文说明。 |

### 4.1 进入与数据

| 玩法 | 说明 |
|------|------|
| **落地动画** | Mobi 自上方落下（mobiDropOffset），约 0.45s 后 land_thud 音效 |
| **画像与记忆** | onAppear 时拉取画像 API（GET /profile/evolution?userId=）、EverMemOS 记忆（各阶段均检索注入 memoryContext；幼年条数较少体现「刚出生没那么聪明」，青年/成年更多）；结果写入 EvolutionManager、Doubao prepareForRoom（persona + memoryContext + stage + languageHabits + confidenceDecay） |
| **语音连接** | Doubao 连接后即可与 Mobi 实时语音对话；用户说话会记录并参与记忆存储与画像；AI 回复会存回合到 EverMemOS |

### 4.2 与 Mobi 的肢体交互

| 玩法 | 说明 |
|------|------|
| **戳击（Poke）** | 点击 Mobi 身体 → spring 形变、squeak 音效、大脑 receiveTouchPoke；行为上报 poke；1s 节流 |
| **长按抚摸** | 长按 Mobi → 行为上报 long_press；大脑可参与（依阶段） |
| **拖拽** | 拖拽 Mobi 可移动其位置；松手后位置保持；拖拽过程 squash-stretch、lookDirection 跟拖拽；行为上报 drag |
| **灵器（Soul Vessel）点击** | 点击胸口灵器 → squeak、行为上报 vessel_tap |
| **灵器长按** | 长按灵器 → 弹出 Soul Sync  sheet，展示 personalitySlotProgress（人格槽同步率） |

### 4.3 语音与大脑

| 玩法 | 说明 |
|------|------|
| **实时语音** | 用户说话 → STT → Doubao 回复 → TTS；Mobi 口型、眨眼、lookDirection 朝用户（listening 时）；说话时 arousal/attention 等参与呼吸与眼神 |
| **Seeking** | 沉默超过约 12s 且本会话未触发过 → 发 seeking 指令，Mobi 主动说一句话（阶段 + confidenceDecay 区分话术）；行为上报 silence_interval |
| **平行陪伴** | 沉默 >3s 且非 seeking/说/听时，Mobi 进入 idle 状态（轻微摇摆）；沉默 >60s 时每 3 分钟 glance（眼神短暂偏转） |
| **大脑状态** | arousal / attachment / curiosity / comfort / energy / attentionLevel 等随 voice、touch、silence、keyword 更新并 decay；驱动 breathScale/breathFrequency、lookDirection、isSeeking、isStartled 等 |

**Room 身份与阶段话术**：Doubao 的 system prompt 与 Trojan Horse 按 Room/Anima 分支注入身份。Room 下问「你是谁」时，Mobi 回答「我是 Mobi，刚来的 / 你给我的样子 / 你的伙伴」等（依 newborn/child/adult 阶段）；Anima 仍答「passing consciousness」。阶段话术由 MobiPrompts.stageGuidanceBlock 注入：幼年 1–2 句简短、本能；青年可引用记忆、好奇活泼；成年引用共同历史、沉稳关怀。**命名仪式**：破冰完成后 newborn 可进入命名仪式，Mobi 问「我不记得我叫什么了…Mobi？那是个好名字吗？」；用户确认后 Mobi 说「好，那我是 Mobi，你是 Star。」**Newborn 学说话阶梯**：newborn 早期（0 铭印且未听过用户）**静默期 mute**：Mobi 不说话，仅对触摸/语音做肢体/眼神/拟声反馈；听过用户后进入 **gibberish** 期，发出 ba/bo/nya 等回应；用户对同一词重复 2–3 次 → 可变奖励（2 次 30%/3 次 70%/4 次 100%）→ **模仿成功** → 铭印 + 庆祝浮层「Star 教会了我：X」；≥1 铭印后进入简单中文。**Child 小孩话术**：child 阶段说小孩那种话（短句、好奇、playful，呀/呢/哇）。**Adult 伙伴话术**：adult 阶段像正常伙伴。

### 4.4 进化与人格槽

| 玩法 | 说明 |
|------|------|
| **人格槽（灵器瓶身）** | 由画像 API 的 slotProgress（或 Mock）驱动，7 格填充；进化阶段（newborn/child/adult）由画像 lifeStage 驱动，只进不退 |
| **槽进度变化** | 当 slotProgress 上升时灵器有「激动」动画（vesselAgitated）；达 100% 且未满溢过 → **先弹 Kuro 进化考核**（进化许可按钮）→ 确认后满溢序列：cracks → burst → merge → done，之后 markVesselOverflowed |
| **进化解锁** | colorShift（青年后主色可变为粉红）、coffeeCup（青年后房间内咖啡杯 doodle）；由 EvolutionManager.hasUnlocked 与 evolution.effectiveStage 控制 |
| **置信度衰减** | 画像返回 confidenceDecay 时，seeking 等话术可带「拿不准你」类表达；阶段不变 |

### 4.5 记忆与日记

| 玩法 | 说明 |
|------|------|
| **对话记忆** | 每轮用户发言 + Mobi 回复完整后，写入 EverMemOS（user/assistant 两条）；进入 Room 时按阶段检索注入 memoryContext（幼年较少条、青年/成年更多），并将最近铭印（最多 8 条）拼入同一块，Mobi 可引用「你上次说过…」以及「Star 教会了我…」；幼年话术简短/本能，体现刚出生没那么聪明。 |
| **昨日日记** | 右上角书本按钮 → MemoryDiaryView：拉取昨日 0:00–24:00 EverMemOS 检索结果，展示 date + bullets + sentiment；下方「Star 教会了我」区块展示铭印记录（ImprintService），最新一条有背景高亮与 spring 动画；无结果或未配置 Key 时回退 mock 文案或「暂无记录」 |

## 六、行为上报与画像（后台）

以下行为会写入 EverMemOS（sender=mobi_behavior 或对话 sender），供画像服务消费；用户无直接 UI，但影响画像完整度与进化：

- session_start / session_end（进入/离开 Room）
- poke / long_press / drag / vessel_tap / vessel_long_press
- silence_interval(durationSeconds)

画像服务（backend/ 或配置的 PROFILE_EVOLUTION_BASE_URL）按 userId 拉取记忆条数并计算完整度 → lifeStage、slotProgress、dimensionConfidences 等，客户端拉取后驱动人格槽与进化阶段。

## 七、当前玩法一览表

| 阶段 | 用户可做 | 系统反馈 |
|------|-----------|----------|
| **登录/注册** | 每次启动先选 ID：输入名字注册或选择已有账户登录 | 进入对应 ID 的 Anima 或 Room，按账号隔离 |
| **Anima** | 与 Orb 语音对话 15 轮、触摸 Orb、长时间沉默被 Lure | Orb 形态/颜色变化、膝跳反射、转场视频 + Cosmic Sneeze、诞生 DNA+人设 |
| **Room** | 与 Mobi 实时语音、戳/长按/拖拽 Mobi、点/长按灵器、打开日记、沉默被 Seeking；**长按左下角巢穴** 召唤 Kuro（设置/重置）；22 点后进入触发晚安；精力耗尽时 Kuro 持账单、可购买能量 | 口型/眼神/呼吸、平行陪伴（idle 摇摆、glance）、铭印日记、晚安 flow、人格槽与进化、Kuro 进化考核后满溢动画；精力耗尽时 Mobi 瘫软 |
| **调试** | Debug 浮层：跳轮次/跳阶段、T/E/S、Reset、设置 | 快速测试不同阶段与连接状态 |
| **设置** | 改 ID/用户名、切换账户 | 多账号数据隔离、重新登录 |

以上均为**当前已实现并可实际玩到的内容**；未实现或待扩展的（如 adult 阶段话术细化、language_habits 注入、Fact 粒度等）见 [MVP-Phase-Plan](MVP-Phase-Plan.md) 与各专项文档。
